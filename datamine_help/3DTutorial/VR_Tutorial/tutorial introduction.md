# Welcome to the 3D Visualization Tutorial

![](../Images/vrt_BlockModels1.gif)

## Introduction

This tutorial introduces you to key software features and procedures used to set up and view data in a simulated virtual environment, and includes the following sections:

  * Getting Started

  * Loading Data

  * Preparing Data for 3D Visualization

  * Navigating and Viewing Data

  * Designing with Locked Sections

  * Working with Block Models

  * Draping Texture Images

  * Environment Settings

  * Adding 3D Objects

  * Creating Drive and Flight Paths

  * Simulations and Sequenced Animations

  * Using 3D Grids

This tutorial is applicable to the following applications only:

  * Studio RM version 1 or later

## Prerequisites

This tutorial assumes the following items are in order before you start:

  * You have a basic working knowledge of your application, including displaying ribbons, creating a new project, digitizing strings and saving files. It is not necessary to have a working knowledge of open pit design in order to follow this tutorial.

  * You have installed the latest version of your application together with the standard data set which is located in the folder C:\Database\DMTutorials\Data. Files from this set will be used throughout the tutorial exercises. If you do not have access to these files, contact your Datamine support representative.

## Using this Tutorial

When you open the tutorial, you will see a list of available topics. If you want to hide or show this list, click Hide or Show on this Tutorial's Help toolbar

This tutorial is arranged so that it, in addition to a few general pages, contains most of the tutorial material in principles and exercise pages:

  * **Principles** \- will give an overview of the principles associated with a particular topic and provide useful background information.

  * **Exercises** \- provide exercises with step-by-step instructions on how to perform a particular tasks, using existing or new tutorial data. There may be one or more exercise pages in a group, typically arranged in a logical sequence. It is strongly recommended to run through the exercises in the order displayed in the Table of Contents
