# Data Table Format  
  
To access this screen:

  * From the [Data Object Manager](<Data%20Manager%20Dialog.md>), select Data Table Options.

Format the view of the data columns shown in the Data Object Manager's [Data Table](<Data%20Object%20Manager%20-%20Data%20Table.md>) tab.

Activity steps:

  1. Open the **Data Object Manager**.

  2. Select a loaded data object.

  3. Select the **Data Table** tab.

  4. Choose if **column resizing** is permitted or not:

     * If **Allow column resize** is **checked** , data table columns can be resized by dragging and dropping.

     * If **Allow column resize** is **unchecked** , table column widths are fixed.

  5. Choose if **warnings** are displayed:

     * If **Disable warnings** is **checked** , warnings relating to incorrect syntax are reported instantly in the Data Object Manager.

     * If **Disable warnings** is **unchecked** , warnings are not issued if table syntax errors occur. They may be reported by other functions later.

  6. Choose which data to display:

     * Toggle **Show Hidden Columns** to manage the display of typically hidden table columns.

     * Toggle **Show System Columns** to control the display of system data columns.

     * Toggle Show XYZ Columns to show or hide coordinate attributes.

     * Toggle Show System Objects to show or hide system data objects in the **Loaded Data Objects** list.

  7. Choose **Number Formatting** options:

     * By default, all data is right-justified. If **Right-justify numbers** is unchecked, numeric data is left-justified.

     * **Use thousands separator** determines if a comma (,) is shown as the separator between thousands in reported numeric values.

     * Specify Digits after decimal controls the resolution of the numbers displayed in the **Data Object Manager** table view. It does not alter the precision of the underlying data values and won't affect how numeric data is saved.
