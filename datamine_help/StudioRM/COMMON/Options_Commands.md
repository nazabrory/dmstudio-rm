# Options: Commands

To access this screen:

  * [Options](<Options.md>) screen **> > Commands**.

Configure settings relating to undo and redo actions.

Note: Storing many undo levels, or undo information for a large amount of data may consume a lot of memory. Consider disabling the undo function for commands involving large amounts of data in order to conserve memory and speed up processing.

Memory currently being used to store undo information can be freed at any time by clicking Remove Undo Information.

To configure your undo and redo system behaviour:

  1. Display the **Commands** screen.

  2. Choose if undo data is stored or not, using Store command undo information:

     * If **checked** , undo information is stored, and actions can be undone and redone.

     * If **unchecked** , undo information is not stored and undo and redo actions cannot be performed.

  3. To remove any previously stored undo information (also known as the "undo stack"), click **Remove Undo Information**.

  4. Click **OK**.
