# Options: Environment

To access these screens:

  * [Options](<Options.md>) screen **> > Environment**.

Set up general properties controlling how your application is displayed. There are several categories on environment options, accessible by menu entries on the left.

## General Options

Options displayed here control how your application will appear the next time it is started.

  * On Startup Select the behaviours you prefer when starting your application:

    * _Show Start Page_ By default, the Studio **Start** page displays. 

Tip: Customize the page that displays using **Web Browser** options.

    * _Load Last Project_ Reopen the most recent project.

    * _Show Open Project_ Display a file browser to pick a project to load.

    * _Show Project Wizard_ Display the Project Wizard to create a new project.

    * _Show Empty Environment_ Do nothing after launching your product.

  * Recent Lists If a **Start** page is displayed, choose the number of items the **Project list contains**. Up to 10 projects can be listed.

  * Tip of the Day No longer used.

  * Start Page Choose if live information is displayed on your **Start** page, using the **Attempt to load online content** toggle.

    * If **checked** , an attempt is made to display a live news page in your **Start** window. If no Internet connection is available, offline content displays. This attempt is made when you click **Apply**.

    * If **unchecked** your **Start** page shows only offline (installed) information regardless of Internet connectivity.

Note: Online content is recommended as it advertises information about the latest available release of your product.

  * Help Files Choose between offline and online help files. 

Note: These settings and the **Start Page** settings (see above) are completely independent.

    * **Offline help only** Installed help is displayed when requested, using the Microsoft Compiled HTML Help format. 

    * **Online help if possible** If a connection with the target online help page can be established, display it in a web browser. If no connection can be made, offline help displays instead.

**Tip** : Access Datamine's online help library at any time using a web browser, by accessing **docs.dataminesoftware.com**.

## Folders Options

Specify the default location of new projects.

## User Information Options

The details associated with project and data;

  * User ID Specify or edit an identification string relating to the current user.
  * Name The name to associate with projects and data.
  * Organisation Specify or edit the name of your organisation.
  * Address Specify or edit the postal address of your company.

## Web Browser Options

There are two web browser pages embedded within your application which can be customized according to your preferences:

  * **Default Start Page** Your application has a default start page but you can replace this with any other web page you desire by using the browser.
  * **Default Customization Page** Choose the default Customization control bar script, say, to show your corporate branding, or to automatically load a template script.

**Tip** : Scripts can be run automatically when a project opens. See [Project Settings: General](<Project%20Settings_General.md>).

## Cursor Tooltips Options

Use this tab to configure if cursor tooltips are displayed and if so, the font and font size to be used.
