# Data Selection Control Bars

Your application uses a number of control bars for navigating data. Each serves a distinct purpose; some relate to the project as a whole whilst others relate to specific windows or to data which is loaded into memory. Each of the control bars may be hidden, docked or floated, and may be grouped with other control bars according to requirements and taste. See [Customizing Control Bars](<Customizing.md>)

Not all tabs may be visible on your system. They can be shown and hidden.

The items described on this page relate to control bars that deal with the selection of data, and the accessing of data related functions using context menus. 

**Tip** : rearrange the order that the tabs appear in on the screen, to suit your needs, by dragging them (hold down the left mouse button when over a tab) to a new position in the tab group.

### Project Files Control Bar

The Project Files bar lists all the files currently associated with your project and categorizes them according to the nature of the data: points, strings, block models and so on.

This is the tool for performing basic file functions such as opening a file (in the table editor), loading files into memory, copying and exporting files. These commands are available through the right-click context menu.

See [The Project Files Control Bar](<Concept_Project%20Files%20Control%20Bar%20Overview.md>).

### Sheets Control Bar

The Sheets bar can be used to control loaded 3D objects. It expands the trees under all windows to reveal the 3D objects and their overlays. It provides the means to show or hide overlays (using the check boxes) and it provides access to display formatting through the context (right-click) menus.

###  Loaded Data Control Bar

The Loaded Data bar is hidden by default. Most of the commands previously accessed in the Loaded Data bar in Studio 3 have been migrated to the Sheets control bar (see above). You can still use the Loaded Data bar to manage non-visual in-memory tables.

The Loaded Data control bar also provides a view of current loaded non-visual tabular data as a listing of tables with field names. 

See [Loaded Data Control Bar](<Loaded%20Data%20Control%20Bar.md>).

### Customization Control Bar

The Customization control bar is a web page in which scripts can be run.

See [Customization control bar](<customization%20window.md>).

### Quick Filters Control Bar

The **Quick Filters** bar allows filters to be generated quickly and interactively for multiple objects based on unique values stored in their data, or based on values in preset ranges. Filters can be saved and applied to other loaded data objects.

See [Quick Filters control bar](<QuickFilterLegendDialog.md>).

### Holes Control Bar

The Holescontrol bar is used to access loaded drillhole-specific information.
