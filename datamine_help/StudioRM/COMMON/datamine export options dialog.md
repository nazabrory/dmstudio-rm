# Datamine Object Export

To access this screen:

  * From the [Data Export](<ExportTable.md>) screen, select the _Datamine_ category and any Data Type and click OK.

Export a file using Data Source Drivers. The title of the screen displays the type of data being exported (such as "Datamine Block Model").

The following options are available:

  * **Output Filename** The path that will be used to store the exported file. Either enter a file location into the field or click the ellipsis button to browse for a file location.

  * **Description** A brief description of the file to be exported (optional).

  * **Use Datamine Project File?** Specify whether you wish to export the object to an existing project file. If selected, you are able to browse for a project file using the Ellipsis button.

  * **Extended Precision** Saves the resulting file in normal or extended precision mode. If selected, files will be exported in double precision format (integers will be exported as floating point numbers). The maximum number of fields in a file is 64 for the single precision files and 256 for the double precision equivalent.

For more information on file precision types, see [Project and File Precision](<Project%20and%20File%20Precision.md>).

  * **Select Fields** Select the fields you wish to include in the resulting output file. By default, all fields will be included. Clicking this button displays the Select Fields dialog, which allows you to select or clear field check boxes.

  * **Rename Fields** Displays the Rename Fields screen, allowing you to rename the descriptions for each data column within the exported file.

  * **Datamine Color Field** If you wish to specify a colour field for the exported file, select a field from the drop-down list.
