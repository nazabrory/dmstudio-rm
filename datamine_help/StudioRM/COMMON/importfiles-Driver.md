# Import Project Files by Driver

Import data created by a 3rd party mining software vendor, one file at a time.

Once a file has been selected, you will need to specify additional parameters as part of the data import process. These settings depend on the data format you are importing. For example, if importing a CAD file, you will see different import screens than a BMF block model file. 

Note: Import screens are often provided as part of a 3rd party driver package, but in some cases we have customized these to provide a more seamless experience when bringing data into Studio products.

When a file is imported, it is first subjected to the functions of the relevant data source driver, which prepare the file for loading. In effect, they 'translate' the incoming file into a format recognizable by your application. 

Once all driver-specific screens are completed, the final **Import Files** screen displays. See [Import and Load Data](<Concept_Importing%20and%20Exporting%20Data.md>) for activities.

To import a single file to the project:

  1. **Data** ribbon **> > Import >> Add to Project >> Add to Project (by driver)**.

The **Data Import** screen displays.

  2. Define the import driver and type. See [Data Import](<data%20import%20dialog.md>).

  3. Click **OK**.

  4. Select a source data file.

  5. Click **Open**.

  6. Complete the driver-specific screens. The contents of these screens are unique to each driver type. Click **Help** or press F1 for more information on these screens.

  7. Click **OK**.

Importation of all selected files completes.
