# VR Object Type Properties

Properties which are common to objects of the same type (model, scale, lights, sound, offsets and movement) can be viewed and edited with the Object Type Properties screen.

Right-click on the object name in the Object Type folder in the Sheets window (active 3D window sub-folder) and choose Properties. You can also double-click on the object type name to display the Properties screen.

Using the **Object Type Properties** screen, you can configure the following:

  * [Stationary objects](<Objects_Stationary%20objects.md>) such as buildings, equipment and landscape features.

  * [Mobile objects](<Objects_Mobile%20objects.md>) such as vehicles, pedestrians and hover viewpoints.

  * [Simulation objects](<Objects_Simulation%20objects.md>) are objects attached to strings (simulations) and viewpoints attached to strings (fly-thru's).

  * [**Object lights**](<Objects_Object%20lights.md>) such as light bulbs, spotlights and headlights on stationary and mobile objects.

  * [**Object sounds**](<Objects_Object%20sounds.md>) such as engine noises on stationary and mobile objects.

Properties that can be varied between objects placed in the world view of the same object type (position, orientation and attachments) can be viewed and edited using the [VR Object Properties](<Object_Properties_Dialog.md>) screen. 

**Note** : a VR object's _type_ is not the same as an individual _VR object_. An object type is a category of object that contains the global settings for all instances of that object type in the active **3D** window. It is possible to set individual properties for an object, but the settings permitted for types and objects are different.
