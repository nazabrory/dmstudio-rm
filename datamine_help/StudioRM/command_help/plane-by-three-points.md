# plane-by-three-points ("3")

See this command in the [**command table**.](<COMMAND%20TABLE_P.md#plane-by-three-points>)

To access this command:

  * **3D View** ribbon **> > Sections >> 3 Points**.

  * Using the **[command line](<../COMMON/Command_Toolbar.md>)** , enter "plane-by-three-points"

  * Use the quick key combination "3".

  * Display the **[Find Command](<../COMMON/findcommand.md>)** screen, locate **plane-by-three-points** and click **Run**.

## Command Overview

Fully define a 3D section by digitizing 3 points in any 3D window, updating the current section definition. Azimuth and inclination (and central position) are all derived from digitized point positions.

No additional screens display with this command.

**Tip** : right-clicking to snap section points to other data can be useful to ensure the section passes through data of interest.

**Note** : there are separate commands available to define the 3D view _direction_. These options are similar to defining a section ([1 point](<view-by-one-point.md>), [2 points](<view-by-two-points.md>), [3 points](<view-by-three-points.md>)) but in each case, no section definition changes, only the direction from which data is viewed in the target 3D window. See also [Viewing Data](<../COMMON/Interface_Viewing%20Data.md>) and [3D Sections](<../VR_Help/Sections.md>).

Command steps:

  1. Orient your 3D view so appropriate data is clearly visible.

  2. Run the command.

  3. Following the prompt, select (snapping is allowed) 3 separate points in any 3D window (points can be digitized in combinations of windows if required).

The current section definition updates so that is passes through all three points.
