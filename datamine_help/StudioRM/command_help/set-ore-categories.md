# set-ore-categories ("soc")

See this command in the [**command table**.](<COMMAND%20TABLE_S.md#set-ore-categories>)

To access this command:

  * Model ribbon **> >**Manipulate >> Adjust to Ore >> Set Ore Categories

  * Using the **[command line](<../COMMON/Command_Toolbar.md>)** , enter "set-ore-categories"

  * Use the quick key combination "soc".

  * Display the **[Find Command](<../COMMON/findcommand.md>)** screen, locate **set-ore-categories** and click **Run**.

## Command Overview

Define the selected evaluation legend categories as ore or waste.

This command is used when the **Legend Items** option has been selected using the [set-adjustment-method-switch](<set-adjustment-method-switch.md>) command in conjunction with [adjust-string-in-ore](<adjust-string-in-ore.md>). In addition, an evaluation legend must already be defined using the[ **Project Settings: Mine Design**](<../COMMON/Project%20Settings_Mine%20Design.md>) screen.

You are prompted to enter whether you wish a category to be treated as ore or non-ore. The default for each category is the current setting. This specification is used in [string adjustment to an ore outline](<adjust-string-in-ore.md>) and in mine sequencing.

Command Steps:

  1. Run the command.

  2. Click Yes or No for the listed legend item.

  3. Repeat step 2 for each legend category.

  4. Click Yes to accept or No to redefine ore categories.
