# Overview

In this portion of the tutorial you going to be introduced to the methods used to open and save a copy of an existing project file.

## Prerequisites

  * You will need access to the Data Presentation Tutorial start point project file and the tutorial data set, which are located (with a standard installation) under C:\Database\DMTutorials. This folder contains two sub-folders, Data and Projects. The contents of the Data folder will be used throughout the tutorial.

  * If you cannot locate these folders, please contact your Datamine support consultant.

## Link to exercises

The following exercises are available on this page:

  * Opening an Existing Project
  * Saving a Copy of a Project

## Exercise: Opening an Existing Project

In this exercise, you are going to load the existing tutorial start point project file Data Presentation Tutorial.StartPoint.dmproj. The procedure for loading this project file is as follows:  
**Opening a Project File**

  1. Click the  Project button and select  Open Select  File | Open... .

  2. In the Open dialog, browse to the folder C:\Database\DMTutorials\Projects\S3DatPresTut\ProjFiles\Standard.

  3. Select the project file name Data Presentation Tutorial.StartPoint.dmproj.

  4. Click Open:

## Exercise: Saving a Copy of a Project

![note.gif \(1017 bytes\)](../images/note.gif) |  This exercise follows on directly from the previous exercise Opening an Existing Project.  
---|---  
  
In this exercise, you are going to save a working copy of the tutorial start point project file Data Presentation Tutorial.StartPoint.dmproj into a new project folder C:/Database/MyTutorials/DataPres. This project file will be used for storing changes made during the course of the tutorial exercises. This project folder will also be used to store any data files created while working through this tutorial. The procedure for saving a copy of this project file is as follows:  

**Saving a Project File**

  1. Click the  Project button and select  Save As File | Save As... .

  2. In the Save As dialog, create and select a new folder C:/Database/MyTutorials/DataPres.

  3. In the File name field, define the new file name as 'Data Presentation Tutorial.dmproj', click Save.  
  
![](../Images/dp_Saving%20a%20Project%201.jpg)  

![note.gif \(1017 bytes\)](../images/note.gif) |  Any files you create will be stored in this new folder: C:/Database/MyTutorials/DataPres/.  
---|---  
  4. If you are shown the Save Data Set Auto Reload dialog, check that the Save and Auto Reload check boxes are as shown below, click OK.  
  
![](../Images/dp_Saving%20a%20Project%202.jpg)

![note.gif \(1017 bytes\)](../images/note.gif) | 

  * The project file extension indicates what version of Studio was used to created the project:
  *     * *.dmd \- Studio version 2 and earlier
    * *.dmproj \- Studio 3 and later
  * The following project files can be opened and read by your application:
  *     * *.dmd
    * *.dmproj (Studio 3 and Studio RM)
    * *.dhx (Downhole Explorer)
    * *.evr (InTouch)
    * *.epr (Present)
    * *.emproj (Studio EM)
    * *.s5dproj (Studio 5D Planner)
  * The project file can be set to be automatically updated after project changes have been made e.g. importing data. This is set via the Home ribbon's Project | Options commandTools | Options, in the Options dialog, Project tab, Automatic Updating sub-tab. Then, selecting both the Detect New Files... and Detect Files Added... check boxes, enables automatic project updating.
  * For more information on project options, consult your online reference Help project (use the 'stack of books' iconHelp | Contents), or the reference topic (open the Project Options dialog and press <F1> on your Keyboard or click Help).
